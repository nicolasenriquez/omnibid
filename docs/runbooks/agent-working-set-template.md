# Agent Working Set Template

Use this template for long-running or multi-turn work that may survive a context reset.

Keep it short, factual, and tied to repo truth. Do not turn it into narrative.

```md
## Working Set

- Goal:
- Who this serves:
- Decision enabled:
- In scope:
- Out of scope:
- Primary docs:
- Source registry entries:
- Key entities or fields:
- Assumptions:
- Validation:
- Trace / run ID:
- Open questions:
- Next turn:
```

## Notes

- Use the smallest validation path that can prove the change.
- If business meaning or table grain is unclear, stop and write that down here.
- If the change touches data contracts, include the exact source docs and tables.
- If the change touches UI copy, keep raw backend field names out of the user-facing summary.
