## Method

This audit uses the current repo as the source of truth:

- frontend workspace code and types
- backend routers, contracts, and models
- operational and pipeline docs
- domain/business docs
- provided screenshot references as north-star visuals only

Legend for `Tipo de disponibilidad`:

- `available_now`
- `derivable_client_side`
- `derivable_backend`
- `requires_endpoint_extension`
- `requires_new_table_or_view`
- `requires_gold_layer`
- `out_of_scope`

## Readout

- `available_now`: status, region, category, estimated amount, current detail evidence, CSV progress, runs, and summary snapshots.
- `derivable_client_side`: local watchlist and simple urgency badges.
- `derivable_backend`: priority buckets and buyer-history synthesis.
- `requires_endpoint_extension`: ingestion KPIs, API request diagnostics, and other facts that exist in tables but are not yet exposed.
- `requires_new_table_or_view`: documents/bases/anexos, trend series, and any durable view that needs a new read model for performance or clarity.
- `requires_gold_layer`: predictive score, ranking score, forecast, or recommendation output.
- `out_of_scope`: mutable workflow controls, unsupported sidebar destinations, generated narrative synthesis in v1, score editors, API scheduling/replay/reprocess controls, notes, and alert delivery.

Display policy: concept-level absence is rendered as unavailable or omitted; field-level optionality is rendered as `Sin dato` in cards and drawers, and `-` only in compact table cells when the layout requires it.
